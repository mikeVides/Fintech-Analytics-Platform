# FinTech Analytics Intelligence Platform

> **Disclaimer:** The dataset used in this project is entirely synthetic 
> and artificially generated using Python (Faker and NumPy). No real 
> customer, financial, or business data was used. This project was built 
> for educational purposes to demonstrate data analytics and business 
> intelligence skills including Python, SQL, PySpark, dbt, machine 
> learning, and dashboard development.

---

## Project Overview
Customer acquisition costs are rising, loan default rates are unclear, 
and inventory stockout risks are unmonitored. Using Python, SQL, 
PySpark, dbt, and Tableau, I built an end-to-end analytics and business 
intelligence platform on a synthetic FinTech dataset — transforming raw 
messy data into actionable business insights across marketing, supply 
chain, credit risk, and data operations.

## Skills Demonstrated
| Category | Skills |
|---|---|
| Data Analytics | SQL, Python, cohort analysis, A/B testing, forecasting |
| Business Intelligence | Tableau dashboards, KPI development, executive reporting |
| Data Engineering | PySpark, ETL pipelines, Delta Lake, dbt modeling |
| Machine Learning | Logistic regression, MLflow experiment tracking |
| Data Quality | Data profiling, validation, cleaning, documentation |
| Automation & CI/CD | GitHub Actions, pytest, scheduled workflows |

## Business Questions Answered
- Which marketing channel acquires customers at the lowest cost?
- Which campaign generated the highest conversion lift?
- Which products are at risk of stockout in the next 30 days?
- What customer profile has the highest probability of loan default?
- What is the month-over-month trend in revenue and loan volume?
- What percentage of incoming records fail data quality checks?
